Inner Voice / Chrome Capture Slice

One selection. One click. One saved idea.

Highlight text anywhere on the web, right-click, and send it to Idea Store. AI creates a useful title; capture still succeeds when AI fails.

Product Promise

Capture should feel smaller than copying, changing tabs, opening an editor, naming the idea, and saving it manually. V1 does one job only: save selected text into the private Idea Store with a useful title.

01Select web text
02Right-click Save to Idea Store
03Extension sends text to local app
04Server creates title and saves
05Badge shows success or failure

Stable Decisions

Runtime
Manifest V3 extension with one service worker.
Language
Plain JavaScript. No bundler, React, framework, or extension dependency.
Permissions
contextMenus, storage, and http://localhost:3000/* only.
Authentication
Dedicated bearer capture token; never the main password.
Saved body
Selected text only. No page URL, title, tags, or HTML.
Generated title
4-10 words, maximum 80 characters, same language when possible.
Selection limit
3-5,000 characters. Reject oversized input; never truncate silently.
AI timeout
25 seconds, leaving response margin below Chrome's 30-second fetch limit; then deterministic fallback title and save.
Feedback
... while saving, a check mark on success, and ! on failure, with tooltip details.
Failure policy
No queue or automatic retry. User retries intentionally.
Cost limit
Maximum 10 capture requests per minute per server process.
Distribution
Unpacked local extension only for V1.

Ownership Boundary

Chrome ownsSelected text, settings storage, context-menu click, loading/success/error badge.
Server ownsToken validation, input limits, rate limit, OpenAI request, title validation, fallback, SQLite save, safe logs.
OpenAI receivesSelected text only. No URL, page HTML, cookies, token, or browsing history.
Database storesGenerated or fallback title plus exact trimmed selected text.

Never put OPENAI_API_KEY in extension files. Anyone can inspect an installed extension. Paid secrets remain server-side.

Non-Goals

  • No Chrome Web Store publishing.
  • No popup, sidebar, content script, webpage toast, or page scraping.
  • No source URL, tags, screenshots, bookmarks, images, or full-page capture.
  • No retry queue, offline sync, batch capture, or history screen.
  • No extension account system or reuse of the website login cookie.
  • No change to idea body width during this slice; editor-width refinement is deferred.

What This Teaches

  • An extension is a separate client, not privileged website code.
  • Manifest permissions define its power and user trust boundary.
  • A service worker wakes for events and should not own durable state.
  • Paid API secrets belong behind a server endpoint.
  • AI enrichment should fail open to reliable product behavior.
  • A good API contract is built before its second client.
