Build Log

A teaching changelog for the AI notebook. Each entry explains what changed, why it exists, how it was verified, and what it does not prove yet.

Step 3.1: Project Skeleton And Static Notebook Shell

Concept

A product needs a runnable body before it gets real behavior. This slice creates the app foundation and one static screen so we can judge the product shape before adding auth, database, or AI.

What Changed

  • Created a Next.js TypeScript app foundation.
  • Kept plain CSS instead of adding Tailwind.
  • Replaced the default starter page with a static notebook shell.
  • Used the Calm Notes Premium direction as the first visual target.

Files Touched

  • package.json: app scripts and dependencies.
  • src/app/layout.tsx: app metadata and global font setup.
  • src/app/page.tsx: static notebook screen.
  • src/app/page.module.css: page-specific layout and styling.
  • src/app/globals.css: global reset, base colors, and typography.
  • docs/build-log.html: this teaching log.

Why This Exists

This gives us a local app that can be opened, inspected, and improved. It also separates product/design validation from backend work. If the shell feels wrong, we fix it before connecting real data.

How To Verify

  • Run npm run lint to check code style problems.
  • Run npm run build to check TypeScript and production build.
  • Run npm run dev and open the local URL.
  • Inspect whether the page is calm, blue, capture-first, and readable.

What This Does Not Prove

  • No login exists yet.
  • No notes are saved yet.
  • No SQLite database exists yet.
  • No AI call exists yet.
  • No VPS deployment exists yet.

Step 3.1a: Visual Alignment Pass

Concept

A generated design image should be treated as a visual target, not a vague mood. This pass brings the static shell closer to the selected Calm Notes Premium reference.

What Changed

  • Added a small icon library for real interface icons.
  • Changed the page into the mock's main structure: sidebar, editor, and right inspector.
  • Moved search, recent notes, and AI recall into the right-side panel.
  • Added the top editor action row and bottom composer toolbar.
  • Reduced visible writing and replaced letter placeholders with icons.

Files Touched

  • package.json: added @phosphor-icons/react.
  • src/app/page.tsx: rebuilt the static shell structure.
  • src/app/page.module.css: rebuilt the page styling around the selected mock.
  • docs/build-log.html: recorded this visual correction.

How To Verify

  • Run npm run lint.
  • Run npm run build.
  • Open http://localhost:3000 and compare the broad layout to the selected mock.

What This Does Not Prove

  • The UI is not pixel-perfect.
  • The buttons and inputs are not wired to real behavior yet.
  • No auth, database, saved notes, or AI call exists yet.
