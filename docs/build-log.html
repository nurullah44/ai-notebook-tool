Step 4: Authentication Slice

Concept

Auth is the privacy gate for the notebook. Before we save private notes, the app must know whether the browser has a valid session.

What Changed

  • Added a login page with one password for the founder user.
  • Protected the notebook route so logged-out users go to login.
  • Added login and logout API routes.
  • Created a signed HTTP-only session cookie with server-side expiry.
  • Documented required auth environment variables.
  • Marked the Authentication Slice done in the stage map.

Files Touched

  • src/lib/auth.ts: password, session token, and cookie validation helpers.
  • src/app/page.tsx: server-side protected notebook route.
  • src/app/notebook-shell.tsx: client-side notebook UI moved out of the route guard.
  • src/app/login/page.tsx: login screen and wrong-password error state.
  • src/app/api/login/route.ts: password check and session creation.
  • src/app/api/logout/route.ts: session clearing.
  • .env.example: fake auth env var names.
  • docs/ARCHITECTURE.md, docs/SECURITY.md, and docs/DEPLOYMENT.md: stable auth decision, risks, and env vars.

A notebook will contain private text. A fake browser-only password would look like auth but would not protect the app. This slice keeps the secret on the server and lets the server decide whether the notebook can render.

How To Verify

  • Run npm run build.
  • Open http://localhost:3000 and confirm logged-out users see login.
  • Try a wrong password and confirm the login page shows an error.
  • Try the correct password and confirm the notebook opens.
  • Use logout and confirm the notebook becomes protected again.

What This Does Not Prove

  • No notes are saved yet.
  • No database exists yet.
  • No automated auth tests exist yet.
  • No app-level login rate limit exists yet.
  • The prototype uses AUTH_PASSWORD directly instead of a password hash.

Concept

What Changed

  • Renamed the visible product from AI Notebook to Idea Store.
  • Replaced the old shell with three title-first, flippable idea cards.
